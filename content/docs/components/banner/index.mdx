## Props

| Prop | Type | Required | Description |
|------|------|----------|-------------|
| `className` | `string` | ❌ | Additional CSS classes for styling |
| `title` | `string` | ✅ | The title of the banner |
| `description` | `string` | ❌ | The description of the banner |
| `buttonText` | `string` | ❌ | The text of the button |
| `buttonIcon` | `React.ReactNode` | ❌ | The icon of the button |
| `buttonLink` | `string` | ❌ | The link of the button |
| `variant` | `"default" \| "minimal" \| "popup" \| "destructive"` | ❌ | The variant of the banner |
| `style` | `"warning" \| "product-announcement" \| "success" \| "info" \| "error" \| "maintenance" \| "promotional" \| "security" \| "beta" \| "holiday" \| "system-status" \| "feedback" \| "migration"` | ❌ | The semantic style of the banner |
| `autoDismiss` | `number` | ❌ | The time in milliseconds to auto dismiss the banner |
| `onDismiss` | `() => void` | ❌ | The function to call when the banner is dismissed |
| `gradientColors` | `string[]` | ❌ | Array of color strings for animated gradient background |

## Semantic Styles

The banner component now supports semantic styles that automatically apply appropriate colors, icons, and styling for different use cases. This makes it easier to create contextually appropriate banners without manually specifying colors and icons.

### Available Styles:

- **`warning`** - For alerts, maintenance notices, and cautionary messages
- **`product-announcement`** - For new features, updates, and releases
- **`success`** - For completed actions, achievements, and positive confirmations
- **`info`** - For general information, tips, and guidance
- **`error`** - For critical failures and system errors
- **`maintenance`** - For scheduled downtime and infrastructure updates
- **`promotional`** - For sales, discounts, and special offers
- **`security`** - For security notices, privacy updates, and compliance
- **`beta`** - For experimental features and beta programs
- **`holiday`** - For seasonal events and holiday promotions
- **`system-status`** - For real-time system status updates
- **`feedback`** - For requesting user feedback and reviews
- **`migration`** - For important system migrations and updates

### How It Works:

When you specify a `style`, the component automatically:
- Applies appropriate background and border colors
- Adds a relevant icon (unless overridden by `buttonIcon`)
- Uses theme-appropriate text colors
- Maintains accessibility and contrast ratios

### Style vs Variant:

- **`style`** controls the semantic appearance (colors, icons, purpose)
- **`variant`** controls the layout and positioning (default, minimal, popup, destructive)

You can combine any style with any variant for maximum flexibility.

## Gradient Colors

The `gradientColors` prop accepts an array of color strings (preferably in rgba format with alpha transparency). The gradient creates a smooth animated background effect.

### Color Guidelines:
- Use rgba colors with alpha values between 0.5-0.8 for best results
- Colors are automatically filtered for optimal contrast in both themes
- The gradient animation uses a 70-degree angle for a diagonal effect
